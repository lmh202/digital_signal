# 槽楔模型测试数据分析

## 项目概述

本项目用于分析不同松紧度下的槽楔敲击测试数据，通过信号处理和机器学习方法实现松紧度的自动分类。

## 技术方案

### 端点检测（峰值检测+谷底切分方法）

使用**包络+峰值检测+峰到峰谷底切分**方法，解决了固定窗口导致的峰值漏检问题：

#### 核心步骤：
1. **带通滤波**：去除直流和高频噪声（100-15000 Hz）
2. **整流**：取绝对值
3. **低通平滑**：移动平均得到包络
4. **峰值检测**：使用 `scipy.signal.find_peaks` 在包络上检测峰值
   - `prominence`：峰值显著性阈值（相对于最大值 30%）
   - `distance`：相邻峰值最小间隔（0.1秒）
5. **谷底切分**：在相邻峰值之间找谷底作为分界点
   - 在两峰中间区域（50%范围）搜索最小值作为谷底
   - 每个峰从前一谷底到后一谷底为一个完整段
6. **兜底方案**：对最后一个峰或谷底不明显的段，使用衰减阈值（峰值的10%）判断结束
7. **峰对齐截取**：从每段提取固定长度信号，峰值对齐到 1/4 位置，不足补零，不跨入下一峰

#### 关键参数（可调）：
```python
BANDPASS_LOW = 100              # 带通滤波器低频截止 (Hz)
BANDPASS_HIGH = 15000           # 带通滤波器高频截止 (Hz)
ENVELOPE_WINDOW_SIZE = 128      # 包络平滑窗口大小（采样点）
PEAK_PROMINENCE = 0.3           # 峰值显著性阈值（相对值）
PEAK_MIN_DISTANCE = 0.1         # 相邻峰值最小间隔（秒）
VALLEY_SEARCH_RATIO = 0.5       # 谷底搜索范围（两峰中间的50%）
DECAY_THRESHOLD_RATIO = 0.1     # 衰减阈值（峰值的10%）
MIN_SEGMENT_SAMPLES = 256       # 最小段长度（采样点）
SIGNAL_DURATION = 0.03          # 提取的固定长度（秒）
```

### 特征提取（16维）

#### 频域特征（13维）：
1. **主频率**：最大幅值对应的频率
2. **频谱质心**：频谱的"重心"位置
3. **频谱带宽**：频率分布的离散程度
4. **5个频段能量比**：0-2k, 2-5k, 5-10k, 10-15k, 15-25.6k Hz
5. **前5个峰值频率**：按幅值排序的频谱峰

#### 时域特征（3维）：
1. **RMS**：均方根值
2. **峰值幅度**：信号最大绝对值
3. **过零率**：信号符号变化频率

### 分类模型

- **模型架构**：3层全连接神经网络（128-64-32）
- **框架**：PyTorch（带 BatchNorm + Dropout）
- **优化器**：Adam（学习率 0.001）
- **训练**：200 epochs，batch size 32

### 方法对比分析：

#### 固定窗口方法（旧）：
- ✅ 简单直接，以峰为中心固定窗口
- ❌ 窗口之间有间隙，可能漏掉紧密的峰
- ❌ 窗口可能包含相邻峰的干扰

#### 峰到峰谷底切分（新）：
- ✅ **无漏峰**：每个峰都有独立段，峰到峰完全覆盖
- ✅ **自适应边界**：根据包络谷底自然分割，每段恰好一个峰
- ✅ **峰对齐+padding**：峰值固定位置（1/4处），不足补零，不跨峰
- ✅ **兜底机制**：对谷底不明显的段用衰减阈值处理
- ⚠️ 准确率略降可能因信号长度从70ms改为30ms导致信息丢失

## 参数调优建议

如果检测效果不理想，可按以下顺序调整参数：

1. **调整峰值显著性**（`PEAK_PROMINENCE`）：
   - 值太小 → 检测到太多伪峰
   - 值太大 → 漏掉真实峰值
   - 建议范围：0.2 - 0.5

2. **调整峰值间隔**（`PEAK_MIN_DISTANCE`）：
   - 根据敲击频率设置（通常 0.1 - 0.5 秒）
   - 值太小 → 可能把一个峰的振荡识别为多峰
   - 值太大 → 可能把连续快速敲击合并

3. **调整谷底搜索范围**（`VALLEY_SEARCH_RATIO`）：
   - 在两峰中间多大比例范围内搜索谷底
   - 0.5 表示中间 50% 区域
   - 建议范围：0.3 - 0.7

4. **调整衰减阈值**（`DECAY_THRESHOLD_RATIO`）：
   - 方案B兜底：峰值衰减到多少比例时结束
   - 值太小 → 段太长，可能包含下一峰
   - 值太大 → 段太短，丢失衰减信息
   - 建议范围：0.05 - 0.2

5. **调整信号长度**（`SIGNAL_DURATION`）：
   - 当前 30ms，可根据信号衰减速度调整
   - 建议范围：0.02 - 0.08 秒

6. **调整带通范围**（`BANDPASS_LOW`, `BANDPASS_HIGH`）：
   - 根据敲击信号的主频范围设置
